Fundamentals of
Visual User Interface Design

The fundamentals of visual user interface design are fairly straightforward. They can be learned by nearly anyone, applied to nearly any visual user interface, and usually improve the ease in which that interface is consumed. How can such wide-reaching claims be true? Because of one simple fact—we're all human, and for the most part we function in a similar way.

Visual Content Is an Idea

Before learning the fundamental principles and tools to use when designing visual user interfaces, there is a critical concept to keep in mind:

The visual representation of content is most effective when it accurately reflects the underlying conceptual meaning of that content.

This means that a visual user interface is nothing more than a medium being used to communicate and interact with an idea, and that idea exists and can be understood without any visual representation. Therefore, a detailed understanding of the idea is necessary in order to create a visual representation that accurately communicates that idea.

Grouping

One principle of visual user interface design is grouping. Grouping is simply the process of creating visual relationships. When properly used, grouping helps to communicate meaning by giving users visual cues that mirror the content's conceptual relationships. These cues reduce the amount of mental effort used to figure out those relationships and understand the content.

Grouping Cues in Basic Forms

Unfortunately grouping is often overlooked in many aspects of user interface design, and content relationships become either weak or misleading. Weak content relationships directly impact usability and cause consumers to expend more mental effort figuring out what is being communicated. Here's a common example—forms.

Misleading Grouping

Notice how the text describing the form field is closer to the form field above it. Confusion arises when visual relationships do not match the content's conceptual relationships.

Ambiguous Grouping

Even spacing creates no visual relationships and requires users to expend more mental effort figuring out if the text is associated with the form field above or below it.

Clear Grouping

Clear grouping creates visual relationships that accurately reflect the underlying conceptual relationships. In this case the text is related to the form field below it.

Guiding Questions

Sometimes questions asked in different ways can be helpful when sorting out the design of visual user interfaces. Below are a few to get you started.

  1. When elements have strong conceptual relationships, are those elements tightly coupled in a visual group?
  2. When elements have weak conceptual relationships, are those elements loosely coupled in a group or not coupled at all?
  3. Does the visual grouping of elements accurately reflect the conceptual relationships of the content?
